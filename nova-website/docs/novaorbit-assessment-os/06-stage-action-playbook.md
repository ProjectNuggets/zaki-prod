# 06 - Stage Action Playbook

NovaOrbit must always connect maturity to action.

The question is not "How mature are you?" The question is "What should happen next?"

## Stage 1: Ad Hoc AI

Signal:

- Individual AI tools and prompts.
- No shared company memory.
- No agreed deployment boundary.
- No workflow owner.
- No evidence ledger.

Common risks:

- Shadow AI use.
- Sensitive data leakage.
- No reusable workflow value.
- Leadership overestimates adoption impact.

Primary objective:

Move from scattered usage to one owned workflow candidate.

Definitive actions:

- Inventory current AI use.
- Identify sensitive workflows.
- Select 3 workflow candidates.
- Name executive sponsor and department owners.
- Map source systems for the strongest candidate.
- Define minimum AI use policy if none exists.

Best first Nova Nuggets offer:

- NovaOrbit Standard.

Do not recommend:

- Multi-agent implementation.
- Broad custom platform.
- Writeback automation.

## Stage 2: Connected AI

Signal:

- Use cases are visible.
- Systems are known.
- Access is partly understood.
- Pilot candidates can be scoped.
- Governance is emerging but not operational.

Common risks:

- Pilot without production path.
- Access assumptions collapse after kickoff.
- No baseline value.
- Unclear approval owners.

Primary objective:

Convert one candidate into a buildable first workflow.

Definitive actions:

- Select first workflow.
- Define AI app or agent role.
- Validate access path.
- Choose deployment direction.
- Establish baseline metric.
- Define human approval model.
- Confirm owners and RACI.

Best first Nova Nuggets offer:

- NovaOrbit Standard if evidence is thin.
- NovaOrbit In-Depth if stack, data, or compliance complexity is material.

Do not recommend:

- Full production SOW until red gates are closed or explicitly scoped.

## Stage 3: Operational AI

Signal:

- AI runs or can run in named workflows.
- Owners and approvals exist.
- Integrations are scoped.
- Evidence can be captured.
- Governance and controls are becoming operational.

Common risks:

- Quality and reliability not measured.
- Human review becomes informal.
- Run model is underfunded.
- Scaling starts before first workflow is stable.

Primary objective:

Harden the first workflow and prepare scale.

Definitive actions:

- Build or harden the first workflow.
- Define output evaluation.
- Implement run ledger.
- Monitor adoption and exceptions.
- Measure baseline vs post-launch.
- Improve retrieval, prompts, tools, and approvals.
- Decide next workflow based on evidence.

Best first Nova Nuggets offer:

- First Workflow SOW.
- Managed ZAKI agent run support.
- In-Depth diagnostic if the first workflow exists but governance/control is weak.

Do not recommend:

- Cross-department expansion until first workflow evidence is credible.

## Stage 4: Owned AI Workforce

Signal:

- Multiple governed AI workflows are live.
- Infrastructure, memory, agents, apps, and operations are managed as one system.
- The company owns the workflow, evidence, and deployment posture.
- Continuous improvement is part of operations.

Common risks:

- Portfolio complexity.
- Agent sprawl.
- Cost drift.
- Governance fatigue.
- Inconsistent evaluation and monitoring.

Primary objective:

Scale governed AI work capacity across roles and departments.

Definitive actions:

- Build AI workforce portfolio.
- Standardize run ledgers.
- Establish agent/app governance board.
- Create reuse patterns for memory, tools, approvals, and deployment.
- Track portfolio value.
- Create quarterly improvement cadence.

Best Nova Nuggets offer:

- Managed Agents.
- Runtime operations.
- Department expansion roadmap.
- NooX or customer-cloud infrastructure scale-out.

## Dimension-Specific Actions

### Where: Infrastructure & Access

| Score Signal | Action |
| --- | --- |
| 0-3 | Create source-system and deployment inventory. |
| 4-6 | Validate access path, permissions, data boundary, and runtime option. |
| 7-9 | Design implementation architecture and control plane. |
| 10-12 | Scale deployment, monitoring, and reusable access patterns. |

### What: Applications & Agents

| Score Signal | Action |
| --- | --- |
| 0-3 | Define workflow candidates and reject generic chatbot scope. |
| 4-6 | Write first agent/app role and choose app surface. |
| 7-9 | Build first workflow with approvals, evaluation, and fallback. |
| 10-12 | Create reusable agent/app patterns across departments. |

### How: Operations & Impact

| Score Signal | Action |
| --- | --- |
| 0-3 | Name owners and define baseline metric. |
| 4-6 | Build RACI, approval model, ROI case, and governance path. |
| 7-9 | Launch run model, evidence ledger, and improvement cadence. |
| 10-12 | Manage AI workforce portfolio and scale based on evidence. |

## Recommendation Logic

Use these rules in readout:

1. If Where is lowest:
   Lead with access, deployment, and control plane work.

2. If What is lowest:
   Lead with workflow selection, agent role, app surface, and integration depth.

3. If How is lowest:
   Lead with ownership, business case, governance, and run model.

4. If all dimensions are low:
   Recommend NovaOrbit Standard and reject implementation until one workflow is selected.

5. If Where is high but What/How are low:
   The client has infrastructure readiness but lacks a business-operating model. Recommend workflow and impact discovery.

6. If What is high but Where is low:
   The use case is attractive but technical access may block implementation. Recommend In-Depth validation.

7. If How is high but What is low:
   Leadership is serious but the workflow is unclear. Recommend workflow mapping before build.

8. If Stage 3 with no reliability plan:
   Cap scale recommendation. Build evaluation, monitoring, and fallback first.

## First Workflow Selection Rules

Choose the first workflow when it has:

- Clear business pain.
- Repetition.
- Named owner.
- Access path.
- Measurable baseline or acceptable proxy.
- Human approval path.
- Manageable deployment boundary.
- Useful output within 60-90 days.

Reject or defer the workflow when:

- It requires broad transformation before value appears.
- It has no owner.
- It needs writeback before trust exists.
- It relies on sensitive data with no boundary.
- It cannot produce evidence.
- It is chosen because it sounds impressive rather than because it can work.

## Definitive Next-Step Phrases

Use:

- "Build first."
- "Validate access first."
- "Create baseline first."
- "Name owner first."
- "Start read-only first."
- "Use In-Depth before SOW."
- "Do not automate writeback yet."
- "Deploy in customer cloud due to data boundary."
- "Use NooX due to local control and latency."

Avoid:

- "Explore AI opportunities."
- "Continue the journey."
- "Transform the organization."
- "Consider governance."
- "Look into integrations."
