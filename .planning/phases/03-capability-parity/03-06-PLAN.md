# 03-06 Plan: Source Upload, Images, Folders, Archives

## Goal

Close hosted-safe source ingestion parity for ZAKI Learn without exposing local server folder paths.

## Requirements

- LEARN-PARITY-001
- LEARN-PARITY-002
- LEARN-PARITY-010
- LEARN-SEC-001 through LEARN-SEC-010
- LEARN-UX-001 through LEARN-UX-007

## Implementation Steps

1. Consume the learning engine supported-file policy in the ZAKI source panel.
2. Preserve browser folder relative paths in multipart filenames.
3. Route browser folder uploads through the explicit hosted folder upload alias.
4. Verify archive uploads use the hosted archive route.
5. Fix the learning engine so image source files are accepted and processed without breaking chat image attachments.
6. Add safe archive expansion in the learning engine with path traversal and size controls.
7. Add browser and backend tests for create/upload/folder/archive/image behavior.

## Acceptance Criteria

- ZAKI source upload inputs use downstream supported-file policy.
- Normal files, images, browser folder selections, and archives are all reachable from `/learn?view=sources`.
- Server-local linked folder paths remain unavailable in hosted ZAKI.
- Archive extraction rejects unsafe paths and oversized extracted content.
- Chat image attachments remain multimodal attachments, not document text.
