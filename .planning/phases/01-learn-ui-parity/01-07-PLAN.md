## Implementation Steps

1. Browser-walk every Learn route with local frontend, BFF, and learning engine running.
2. Compare visible shape against upstream screenshots/code for the matching module.
3. Record route truth and any gaps in `01-07-SUMMARY.md`.
4. Run a focused code review on `LearningPage.tsx`, `learningApi.ts`, and learning BFF changes touched during Phase 01.
5. Fix any P0/P1 regressions found during the review before completing the phase.
6. Run typecheck, targeted tests, and `git diff --check`.
7. Decide whether Phase 02 security hardening or Phase 03 deeper capability parity should run next based on findings.

## Acceptance Criteria

- Every Phase 01 route renders the intended ZAKI-native upstream-shaped surface.
- Main ZAKI categories remain untouched.
- Learn side-panel behavior remains scoped under Learn.
- No DeepTutor branding is visible in production UI.
- No provider/model/API-key settings are exposed to normal users.
- Findings are recorded with priority and assigned next phase if not fixed immediately.
