# Phase v1.5-frontend Plan 03: WS2 Audit Fixes (C3, C4, C5, M3) Summary

Lifted SystemNoticesStack to ChatArea shell, expanded SSE handler to dispatch 4 additional event kinds as system notices, moved ContextGauge out of its nullalisMode gate, and added a 3-tier pressure-color conic gradient to InputArea.

## What Was Built

### C3 — Expanded SSE Handler (ChatArea.tsx ~line 3988)

Added a new condition block in `readPayloadChunk()` that fires before the `isZakiAgentSpace` text-filter guard. It handles:
- `eventType === "system_notice"` or `payloadType === "system_notice"` (existing semantics, now via `emitSystemNotice`)
- `eventType === "compaction"`
- `eventType === "provider_fallback"`
- `eventType === "connector_stale"`
- `eventType === "multimodal_failure"`

For the 4 direct event types, `kind` is set to `eventType as SystemNoticeKind` and `message` is taken from `payload.message`.

### C5 — SystemNoticesStack Lifted to ChatArea Shell

Created `src/app/components/ui/zaki/SystemNotice.tsx` with:
- `SystemNoticeKind` union type
- `SystemNoticePayload` interface
- `emitSystemNotice()` — calls all registered listeners (module-level `Set<NoticeListener>`)
- `SystemNoticesStack` component — subscribes via `useEffect`, auto-dismisses after 8s, renders dismissible amber strips

Mounted `<SystemNoticesStack className="-mb-2" />` in ChatArea.tsx between the header block and the `<div className="flex-1 relative z-10 ...">` main content scroll container. This placement ensures notices render on **all views** (home, spaces, chat, brain).

ChatView.tsx does NOT contain SystemNoticesStack.

### C4 — ContextGauge Un-gated (ChatView.tsx)

Removed `<ContextGauge data={contextGaugeData} />` from inside `renderNullalisArtifacts()` (which was gated on `hasNullalisArtifacts`, requiring `nullalisMode && botMode`).

Added persistent render as the **first child** inside the `zaki-chat-thread` return div:
```tsx
{contextGaugeData ? <ContextGauge data={contextGaugeData} /> : null}
```
This renders whenever `contextGaugeData` is non-null, regardless of `timelineMode`, `nullalisMode`, or `botMode`.

### M3 — InputArea Conic Gradient Pressure Colors (InputArea.tsx)

Added `zakiContextPressurePercent?: number | null` prop (default `null`).

Computed `pressureColor` with spec-locked thresholds:
```tsx
const pressureColor =
  zakiContextValue <= 50
    ? "#22c55e"   // green
    : zakiContextValue <= 75
      ? "#f59e0b" // amber
      : "#f10202"; // red (brand)
```

Added the context meter button (`data-testid="zaki-context-meter"`) with Tooltip, placed between `<span className="flex-1" />` and the mic button. Conic gradient uses `pressureColor` as the fill color.

The 50/75 thresholds are intentionally different from ContextGauge's 70/90 bar thresholds — per spec.
