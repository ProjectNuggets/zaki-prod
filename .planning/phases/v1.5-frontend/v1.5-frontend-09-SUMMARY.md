# Phase v1.5 Plan 09: BrainComposeModal Summary

In-place slide-up synthesis panel with framer-motion spring animation, gated on 2+ selected graph nodes, posting to POST /brain/compose via useBrainCompose mutation.

## What Was Built

### Task 1: BrainComposeModal component + i18n

Created `src/app/components/brain/BrainComposeModal.tsx`:
- `AnimatePresence` + `motion.div` with spring(stiffness: 320, damping: 28)
- Slides from `y: "100%"` to `y: 0`; exits the same way
- Auto-suggests title from first 1-2 selected node summaries (sliced to 80 chars)
- Resets title/content/error when `open` goes false
- Submit calls `compose.mutateAsync({ title, content, references: selectedNodes.map(n => n.id) })`
- Shows "You synthesized" success indicator for 800ms then calls `onClose()`
- `canSubmit` gates on non-empty title + content, 2+ nodes, not pending
- Error display on catch

Added `brain.compose.*` i18n keys (8 keys) to both `en.json` and `ar.json`.

Exported from `src/app/components/brain/index.ts`.

### Task 2: Mount in BrainPage

Updated `src/app/components/brain/BrainPage.tsx`:
- Imported `BrainComposeModal` and `BrainGraphNode`
- Derived `selectedNodes` by filtering `graphQuery.data?.nodes` against `selectedNodeIds`
- Wrapped graph slot div with `className="relative"` so absolute panel anchors to it
- Mounted `BrainComposeModal` inside graph slot, gated on `selectedNodeIds.length >= 2`
- `onClose` resets `selectedNodeIds` to `[]`
- Removed placeholder comment
