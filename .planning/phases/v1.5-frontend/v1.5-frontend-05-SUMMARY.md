# Phase v1.5 Plan 05: WS4 Session List Wiring Summary

Session row metadata enriched across sidebar and management sheet with mode pill, live dot, lastChannel chip, and approval count badge using consistent brand chip styles.

## What Was Built

**Task 1 (ZakiSessionList):** Replaced the old amber indicator dot on the MessageSquare icon with a proper chip row rendered below the session label. The row contains (left to right): mode pill (uppercase, `bg-zaki-raised`, hidden when null), live dot (`size-2 rounded-full`, green when live, grey when idle, always shown), lastChannel chip (`bg-zaki-raised`, hidden when null), approval count badge (`bg-[#f10202]`, shown only when count > 0), and the timestamp as plain text.

**Task 2 (SessionManagementSheet):** Inserted mode pill, live dot, and lastChannel chip into the existing `flex items-center gap-1.5` header row alongside the laneLabel badge and pending approval text. The existing approval-waiting amber badge was preserved. `flex-wrap` added to header row to handle wider metadata gracefully.

## Field Sources

All four fields read directly from the `AgentSession` type which already had them typed:
- `session.mode` (AgentSessionMode | "background" | null)
- `session.live` (boolean | undefined)
- `session.last_channel` (string | null | undefined)
- `session.pending_approval_count` (number | null | undefined)

No changes to `api.ts` type were needed.
