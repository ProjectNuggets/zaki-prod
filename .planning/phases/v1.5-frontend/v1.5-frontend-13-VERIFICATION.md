## Manual Smoke Checklist (Recommended for Human QA)

1. Visit `/brain` in dev — page renders with Timeline tab as default.
2. Switch to Graph tab — SVG nodes render after spring physics settles (300 iterations).
3. Shift-click 2+ nodes — compose panel slides up from bottom of graph.
4. Enter a title and content, submit compose — success indicator appears for 800ms then panel closes; graph + timeline re-fetch.
5. Sidebar Brain entry highlights when on `/brain` (active state indicator bar + bg-zaki-selected).
6. Dashboard Brain card visible on the ZAKI home screen, showing memory count and animated node cluster.
7. During an agent session with a pending approval: press `y` to approve, `n` to deny (focus must not be in a text input).
8. Sandbox badge visible in ChatArea header breadcrumb row when agent session is active with sandbox enabled.
9. Session list rows (sidebar + management sheet) display mode pill, live dot, lastChannel chip, and approval count badge.
10. Receiving a compaction/provider_fallback SSE event — SystemNoticesStack amber strip appears at top of ChatArea (above all views).
11. Context pressure: InputArea conic gradient turns amber above 50% and red above 75%.
12. ContextGauge bar visible during normal chat (not hidden when timelineMode is off).
13. Mobile breakpoint (< 640px): BrainGraphView renders vertical list fallback instead of SVG.
14. ThumbsDown button appears on assistant messages (via MessageActions) and fires toast feedback on click.
