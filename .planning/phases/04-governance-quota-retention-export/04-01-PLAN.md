# 04-01 Quota Model And Enforcement Matrix

Date: 2026-05-07

## Goal

Make ZAKI Learn quota behavior explicit, surfaced, and enforced at the hosted BFF boundary before requests reach the learning engine.

## Decisions

- Learning has its own `learning` quota surface. It is not folded into `zaki_bot` because Learn includes uploads, generated books, RAG, notebooks, research, solve, visualization, and tutor agents.
- All user-facing DeepTutor capabilities remain available in hosted ZAKI Learn unless they are unsafe in SaaS. Limits scale by entitlement tier instead of removing features.
- Raw provider/model/API-key settings remain operator-managed and are not part of user quota policy.
- Local filesystem folder linking remains excluded from hosted production. Browser folder upload and archive upload are supported; cloud folder connectors are V1.1.

## Enforcement Matrix

| Capability | Free | Student | Personal / access code | Enforcement |
| --- | ---: | ---: | ---: | --- |
| Daily prompt/mutating turns | Runtime setting | Runtime setting | Runtime setting | Enforced by `learning` prompt bucket |
| Max learning request/upload bytes | 25 MB | 100 MB | 100 MB | Enforced by `/api/learning` BFF and raw upload stream limiter |
| Max files per request | 20 | 100 | 250 | Policy surfaced; route-level file count enforcement follows with parser-level accounting |
| Browser image upload | Yes | Yes | Yes | Enabled |
| Browser folder upload | Yes | Yes | Yes | Enabled |
| Archive upload | Yes | Yes | Yes | Enabled |
| Cloud folder connectors | No | No | No | V1.1 |
| Tenant source storage | 250 MB | 2 GB | 10 GB | Planned storage accounting |
| Artifact storage | 100 MB | 1 GB | 5 GB | Planned artifact accounting |
| Generated books/day | 1 | 5 | 20 | Planned daily counters |
| External searches/day | 3 | 25 | 100 | Planned daily counters |
| Concurrent learning sessions | 1 | 3 | 5 | Planned session accounting |

## Implementation Steps

1. Add a single Learn quota policy resolver keyed from trusted ZAKI entitlement state.
2. Decorate `/api/usage/quota?surface=learning` with Learn-specific policy and current daily prompt usage.
3. Enforce plan-specific request byte limits after auth, while retaining the global operator byte cap before auth.
4. Make raw/chunked upload stream limiting use the authenticated Learn policy cap.
5. Add unit coverage for tier resolution, access-code handling, request-size enforcement, and status payload shape.
6. Update the integration spec to close the `learning` vs `zaki_bot` quota decision.

## Acceptance

- Free users cannot exceed the free Learn upload/request cap even if the operator global cap is higher.
- Paid/access-code users resolve to the correct hosted Learn policy.
- The public usage quota endpoint returns policy truth for Learn.
- The code distinguishes currently enforced quota dimensions from planned storage/book/search/session counters.
