# 04-05 Operator Deployment Checklist With Immutable Image Tags

## Goal

Close the last Phase 04 governance gap by turning deployment prerequisites into an operator-visible readiness gate and durable runbook.

## Scope

- Add a backend readiness helper for hosted learning deployment gates.
- Expose a super-admin endpoint for deployment readiness.
- Require immutable image references for ZAKI and the learning engine.
- Require mirrored learning-engine repository and exact source commit pinning.
- Tie readiness to existing retention and disaster recovery gates.
- Document final-user setup versus operator-managed settings.
- Document route/function user-test coverage and UI popup behavior.

## Acceptance Criteria

- Missing auth, learning config, tenant root, image pins, source commit, retention, or DR evidence returns `ready:false`.
- Complete operator configuration returns `ready:true`.
- No secrets are returned by the readiness payload.
- Normal users cannot access the readiness endpoint.
- Route/function user-test plan exists for every Learn route.
- Automated tests cover readiness helper behavior and all current Learn parity browser routes.

## Verification

- `npm --prefix backend test -- --runTestsByPath src/learning-deployment-readiness.test.js src/learning-disaster-recovery.test.js src/learning-retention.test.js`
- `npm run test:e2e -- --project=chromium-desktop e2e/learning-parity.spec.ts`
- `npm run typecheck`
