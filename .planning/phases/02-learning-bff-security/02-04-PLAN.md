# 02-04 Plan: Generated HTML Renderer Safety

## Goal

Prevent upstream/LLM-provided interactive HTML from executing arbitrary scripts in hosted multi-user production unless an operator explicitly enables a constrained renderer.

## Requirements

- LEARN-SEC-004
- LEARN-SEC-005

## Code Truth

- `src/app/components/learning/LearningBookBlockContent.tsx`
- Current risk: interactive blocks can render `srcDoc` with scripts enabled.

## Implementation Steps

1. Inspect all generated HTML/interactive block render paths.
2. Add a safe default renderer policy:
   - scripts disabled by default
   - operator flag required for script execution
   - clear fallback for interactive blocks when disabled
3. Avoid exposing operator policy controls to normal users.
4. Add frontend tests or focused component tests if the repo has a suitable pattern.
5. Browser-check a book block surface if local data exists; otherwise verify by code/test.
6. Update `02-04-SUMMARY.md`.

## Acceptance Criteria

- Generated HTML cannot execute scripts by default.
- Hosted ZAKI production is not less secure than before.
- Operator-managed renderer behavior is documented in code/spec planning.
