# 02-02 Plan: Raw Upload Byte Limits

## Goal

Ensure hosted raw learning uploads cannot bypass request byte caps with chunked transfer or missing `Content-Length`.

## Requirements

- LEARN-SEC-002
- LEARN-GOV-001

## Code Truth

- `backend/src/index.js`
- Existing ingress gate: `checkLearningContentLength`
- Existing raw proxy limiter: `createLearningByteLimitedStream`
- Existing contract tests: `backend/src/learning-bff-contract.test.js`

## Implementation Steps

1. Review raw learning upload routes:
   - knowledge create
   - file upload
   - folder upload
   - archive upload
2. Verify byte counting applies to the proxied stream regardless of `Content-Length`.
3. Add or strengthen tests for missing `Content-Length`, invalid `Content-Length`, over-limit `Content-Length`, and chunked/mid-stream overflow behavior.
4. Ensure failure responses are `413` and do not leak upstream internals.
5. Run backend contract tests and relevant frontend tests if API behavior changes.
6. Update `02-02-SUMMARY.md`.

## Acceptance Criteria

- Raw upload streams are byte-limited before reaching upstream.
- Chunked requests cannot exceed `ZAKI_LEARNING_MAX_REQUEST_BYTES`.
- Oversized requests return a stable hosted `413` payload.
