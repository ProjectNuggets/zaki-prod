# 02-05 Plan: User And Operator Settings Split

## Goal

Verify and close any remaining places where normal users could see or set operator-managed learning infrastructure settings.

## Requirements

- LEARN-SET-001 through LEARN-SET-004
- LEARN-SEC-005

## Code Truth

- `src/app/components/learning/*`
- `src/lib/learningApi.ts`
- `backend/src/index.js`
- `backend/src/learning-bff-contract.js`

## Implementation Steps

1. Search frontend and backend for provider/model/API-key/base URL setting surfaces.
2. Classify settings as:
   - user-managed learning preferences
   - operator-managed infrastructure settings
3. Remove, hide, or gate any user-visible operator-managed settings.
4. Ensure backend strips/rejects operator-managed fields even if the frontend is bypassed.
5. Add focused tests for any changed guard.
6. Update `02-05-SUMMARY.md`.

## Acceptance Criteria

- Normal users can configure learning behavior, sources, tutors, memory, skills, and content settings.
- Normal users cannot configure provider/model/API keys/base URLs.
- Operator-managed settings are enforced server-side, not just hidden in UI.
